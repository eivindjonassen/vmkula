# Documentation Generation

**Input**: Path to a source code file.
**Context**: The feature specification and plan.

## Execution Flow
1.  Read and understand the code in the source file provided.
2.  Analyze each function, class, and type definition.
3.  Generate documentation comments (e.g., JSDoc, TSDoc, Python Docstrings) for each public member.
4.  The documentation should explain the **purpose** of the code, its **parameters**, and its **return value**.
5.  Ensure the generated documentation follows the project's established style, as defined in its constitution.
